Item Type Comment Response
1 General Has the choice of using OI been made? What are the other options? Yes, We tried open installer framework for our TP2 refresh release. Given other limited options that are not stable, the current choice is Open Installer.
2 Clarification 2.2.1/ Does this state that we will need to maintain OpenInstaller ourselves when we could be relying on another community-supported technology? Yes, We have invested and made good progress using Open Installer in the last 2 months. We would like to leverage the work and our learning to our advantage.
3 Additional info 2.2.4/ Administration command line 'asadmin' should be stable. Right, need to fix https://github.com/javaee/glassfish/issues/3931 for instance In the past releases asadmin was marked as unstable interface. Will clarify this from admin team.
4 Document formatting (minor) Duplicate 2.2.4/ sections Thanks, removed.
5 SWAN links Several links point to SWAN (such as sac.sfbay) There were a couple pointing to Sysnet Registration, removed them.
6 Clarification 4.4.1 - Can you describe the expected "Silent install" feature? When the installer program is run with -n <Silent file to be generated>, the installer would not actually perform the installation, but generate the response/statefile that then can be used with -a <Option> to perform the installation in non-interactive(silent) mode. The statefile would expose all the parameters that would otherwise be filled in by user during a regular, interactive installation. A sample statefile is already attached to section 4.5.1.
7 Clarification 4.9 - What do we mean by localized license? Do we have the license translated and approved in the targeted languages or is this solely at the infrastructure level to make sure it can be done after the fact? Yes, the license gets translated. Installer stores the licenses as localizable strings.
