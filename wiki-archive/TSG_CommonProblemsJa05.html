リモートサーバにアクセスできない

アプリケーションサーバーの開始ページをオープンしようとしたとき、初期画面が表示されない。

次の点を調べてみてください:

  • サーバーはローカルからアクセスできるか?
  • プロキシの設定が問題を引き起こしてないか?

サーバーはローカルからアクセスできるか?

  • 説明
    \Web からサーバーにアクセス出来ないけど、ローカルではアクセスできる場合、サーバーは実際には起動しています。
  • 解決法\\\\\サーバーがローカルで動いているか確認します。
    \
    • サーバーがローカルで動いているかを確認する方法
      \
    1. サーバーが起動しているマシンにログオンする
      \
    2. ローカル Web ページを表示する。例えば、8080 がデフォルトポートならば、次の URL にアクセス\\\\\~UWC_TOKEN_START~1278691907363~UWC_TOKEN_END
      \開始ページが表示されれば、リモートからサーバーにアクセスする Web Connection に問題があります。開始ページが表示されなければ、サーバーが起動しているか確認してください。

プロキシの設定が問題を引き起こしてないか?

  • 説明
    \サーバーが起動しているホスト (localhost) からはサーバーに直接アクセスできるようにします。(例) デフォルトポート 8080 を使っている場合:\ 
    http://localhost:8080/
  • 解決法
    \サーバーホストマシンがプロキシを通して Web と接続している場合、localhost で動作しているサーバーインスタンスにアクセス出来ない場合があります。これを解決するには、次のうちの１つを試してみてください:\
    • localhost にアクセスする場合、プロキシサーバーを迂回するようブラウザを設定する。設定方法は、それぞれのブラウザのヘルプやマニュアルなどをご覧ください。
      \
    • 完全指定ホスト名や IP アドレスを使う。(例)\\\\\~UWC_TOKEN_START~1278691907365~UWC_TOKEN_END
      *注意:* localhost マシンのホスト名やドメイン名を探す方法:\
      • Microsoft Windows の場合: デスクトップで、「マイコンピュータ」をマウスの右ボタンでクリックし、ポップアップメニューから「プロパティ」を選択します。システムプロパティダイアログが表示されるので、「ネットワーク」をクリックし、コンピュータ名を確認します。,
        \
      • Solaris および Linux の場合: コマンドプロンプトで hostname と入力します。
