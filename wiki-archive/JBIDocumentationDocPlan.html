<p></p><h2><a name="overview"></a>JBI Integration Overview</h2>
JBI Integration will be documented in the Application Server documentation set insofar as users interact with JBI as part of the overall appserver. The following user views will be documented:
<ul>
<li>Admin Guide will add conceptual information needed to understand the JBI integration in the AppServer.
</li><li>Admin Console will document the JBI node screens and standalone-instance JBI tab (inline and Online help).
</li><li>Developer's Guide will add JBI integration info (JSR 208) from a developer's perspective.
</li><li>Reference manual will be updated with the new JBI asadmin commands.
</li><li>Installation Guide updated as necessary to include any JBI related screens (covers install of AppServer NOT glassfish).
</li><li>QuickStart guide will be updated with any JBI setup as necessary (not required for Milestone 2)
</li><li>Troubleshooting guides will be updated with JBI troubleshooting tips.
</li><li>Error message online wiki catalog will add any JBI related error messages.
</li><li>Release notes will include any JBI known issues.</ul>
<p>
All AppServer documents will reference the Open ESB opensource website and the Open ESB documentation.
</p><h3>JBI Integration and AppServer Profiles</h3>
<p>In this release, AppServer Editions are replaced with Profiles. As a
result of this change, content that applies only to a particular profile will carry
a statement to that effect. Only one version of the bundled docs (e.g.
OLH page), will be produced.
</p>

<p></p><h2><a name="criteria"></a>Release Exit Criteria for Docs</h2>
<p>
</p><ul>
<li>Monthly glassfish wiki postings: pre-beta quality; snapshot of docs in their current state.
</li><li>Beta: Full doc set, beta quality, no unwaived P1 or P2 bugs.
"Full doc set" means that no guides will be missing. "Beta quality"
means that the scope of a document might be cut in order to meet the
schedule deadlines. The tech writers will work with the engineers to
determine which topics are the highest priority.
</li><li>RR: Full doc set, no unwaived P1, P2, or P3 bugs.
</li></ul>
