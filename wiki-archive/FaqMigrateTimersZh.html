如何备份持久性计时器和 JMS 队列及主题？

（大部分摘自此线索）

如 http://docs.sun.com/app/docs/doc/819-7759/aerbz?a=view 中所述，通过使用确认、事务和持久性消息传送，可以实现可靠的消息传送。

OpenMQ 中提供的持久性服务在 http://docs.sun.com/app/docs/doc/819-7759/aercs?a=view 中进行了说明。

使用 Sun Java System Application Server 9.1 及更高版本，通过高可用性数据存储 HADB，我们现在在 JMS 路径方面也获得了高可用性，如 http://docs.sun.com/app/docs/doc/819-3679/abdbx?a=view 中所述。

可以将 JMS 持久性消息配置为位于任何数据库（例如，Oracle 或内置 HADB（如果在"企业"配置文件中使用的是 Sun Java System Application Server））中。如果在 GlassFish 中使用 JMS 提供者，并将文件系统作为消息存储，则必须确保该文件系统具备高可用性。

计时器元数据可以保留在任何支持 XA 的数据源中。因此，任何可靠的数据库都可行。请遵循供应商的复制/可靠性准则。
