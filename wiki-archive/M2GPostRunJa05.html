GlassFish アプリケーション・サーバー に、ビルドされたファイルを配備する

配備可能なユニットへ移行されたアプリケーションをビルドした後、 GlassFish アプリケーションサーバーへ、配備することができます。 :

  • 入力が、 EAR 、 WAR 、あるいは、 JAR ファイルの場合、マイグレーションツールは、移行プロセスを完了させて、 <output_directory>/Input_Archives/asmtbuild ディレクトリでビルドされた EAR ファイルを配置する、ビルドスクリプトを自動的に一度だけ呼び出します。たとえ移行が成功したとしても、この EAR は、唯一ビルドされるでしょう。

そのビルドは、自動的に生成されないかもしれません。たとえ :

  • いずれにせよひとつのファイルが移行に失敗する。
  • [ インストールとセットアップ |M2GInstSetup] が不適切である。
  • Solaris において、移行されたソースコードが配置され、そして、 myunit{{.ear}} が、配備可能なユニット名であり、 /migrated_output が、出力ディレクトリである場合 :

/migrated_output/asmtbuild ディレクトリに移動し、
asadmin コマンドを実行し、アーカイブを配備します。 :

% asadmin deploy -u <username> --passwordfile <passwordfile> -H <hostname> -p <adminport> myunit.ear
  • Windows において、移行されたソースコードが配置され、そして、 myunit.ear が、配備可能なユニット名であり、 C:\migrated_output が、出力ディレクトリである場合 :

C:\migrated_output\asmtbuild ディレクトリに移動し、
asadmin コマンドを実行し、アーカイブを配備します。 :

PROMPT> asadmin deploy -u <username>
--passwordfile <passwordfile> -H <hostname>
-p <adminport>
myunit.ear

留意事項 - 生成されたビルドスクリプトに指定されている deploy 引数によって、アプリケーションのビルドと配備と言う複数のステップを、単一のステップに結合させることができます。このオプションを利用するために、ビルドスクリプトの生成 (Solaris と Linux 上でのマイグレーションツールの呼び出し、 Windows 上でのマイグレーションツールの呼び出し ) を実行することで、ステップを修正するには次のとおり :
Solaris では :

% sh build_ear.sh -Dw=<passwordfile> deploy

Windows では :

PROMPT> build_ear -Dw=<passwordfile> deploy

ビルドスクリプトは、次のデフォルト値を引き継ぎます。 :

  • ユーザー名は "admin" です。
  • ホスト名は "localhost" です。
  • 管理ポートは "4848" です。

次のコマンドにより、これらデフォルト値を継承できます。 :
Solaris では :

% sh build_ear.sh -Du=_<username>_ -DH=_<hostname>_ -Dp=_<portno>_ -Di=_<instancename>_ -Dw=_<passwordfile>_deploy

Windows では :

PROMPT> sh build_ear.sh -Du=_<username>_ -DH=_<hostname>_ -Dp=_<portno>_ -Di=_<instancename>_ -Dw=_<passwordfile>_deploy

マイグレーションツールが、移行レポートファイルを生成した場合、追加の引数、 -Dp=y を、 deploy 引数に加えて指定する必要があります。
