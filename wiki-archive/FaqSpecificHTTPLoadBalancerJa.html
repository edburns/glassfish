web サービスをクラスタに配備したいのですが、特定のロードバランサプラグインが必要ですか？

users@glassfish メーリングリストより:

質問 Web サービスを開発していて、GlassFish v2 を使って２台のマシンでクラスタにしています。
クラスターに Web サービスを配備したいと思ってます。それらは web application (.war ファイル) として
配備されます。HTTP ロードバランサプラグインをインストールする必要はありますか？HTTP ロードバランサプラグインは
クラスタノード間のジョブだけをロードバランスするのでしょうか、それともクラスタに配備されているなら Web サービス
も自動的にバランスされるのでしょうか？

回答 Alexis: クラスタとデータ複製機能には特別なロードバランスソリューションは必要ありません。
GlassFish に入っているソフトウェアロードバランサを使えます（そして GF 管理コンソールから管理できます）。
あるいは、
Prashanth のブログエントリ
に書かれているような、
いずれの場合も、クラスターの前に何らかのロードバランサを置いてください。
