Admin Console Demo for MS3

Clustering management support including  Listing/Create/Delete of Nodes,  Clusters and Standalone instances.   Start and Stop of  instances and clusters.    Full Application Deployment support targeting any clusters and instances.

Demo

Note:  This demo is shown using just 1 machine,  so the additional Nodes,  standalone instances and clusters are all running  on localhost.   If you have another machine that has configured SSH to use key (certificate) authentication,  you can create the additional Node by supplying all the information on the screen.   The rest of the demo will be the same.

Setup:  

  1. Install GlassFish 3.1  Open Source Edition  MS3
    1. Download http://dlc.sun.com.edgesuite.net/glassfish/3.1/promoted/glassfish-3.1-b12.zip&nbsp;
    2. unzip
  2. Start up Domain 
    1.  glassfishv3/glassfish/bin/start-domain domain1
  3. Start up Admin Console
    1. go to  http://localhost:4848

1.  Application Deployment when there is only DAS

The demo shows the deployment and applications related screens when there is only DAS.   This is to have the same developer experience as in v3.

  1. From the tree,  go to Applications tree node,  click Deploy button
  2. Notice that there is no need to select any Target,  now select hello.war and press Ok
  3. Notice that hello.war is deployed and the Enabled column showed the 'check mark' indicating the app is enabled
  4. click on 'hello',  notice that there is only 2 tabs:  General and Descriptor

2.  Listing and Creating Nodes

This shows how you can create a Node and listing out what is  available in the domain.

  1. From the tree, go to 'Nodes'  which lists out the Nodes that is available.   
  2. click New to create a new Node.   Fill in the following info and click OK.  (Note: if you have other machine with SSH setup, enter the information appropriately.
    1. Name:    Another
    2. Node Host:    localhost
    3. Installation Directory:   /export/glassfishv3
  3. The new Node,  Another,  is successfully  created and listed. 

3. Clusters management

Using the Node created from previous step,  you can create cluster, and its instances.   Start the instance,  delete a cluster etc.

  1. Click on the  'Clusters'  Tree node.    click on New...  to create the cluster
  2. Enter the following information:  
    1. Cluster Name:    clusterABC
    2. Configuration:    default-config
    3. Click on New... button to create the instances.   Create 2 instances:
      1. Instance  Name:   ABC-1 ,   Node:    localhost
      2. instance Name:  ABC-2,      Node:  Another
  3. click Ok and the clusters table will show the newly created cluster
  4. Examine the cluster by clicking on the 'clusterABC'  link. 
  5. Click on Instances Tab,    this shows the instances,  configuration and status etc.
  6. Select  ABC-2 ,   click Start button.    
  7. Note that the status for ABC-2  has changed to Started
  8. Click the Clusters tree node again.  Notice the status has changed also.
  9. Create additional cluster,  XYZ,   with 1 or 2 instances
  10. Select this new cluster and press Delete button
  11. Note that the instances,  and the cluster is deleted together.   (In CLI,  you need to create the instance first, then the cluster) 

4. Stand-Alone Instances management

Using the Node created previous,  this shows you how to create/delete/start standalone instance.

  1. Select 'Stand-Alone Instances'  tree node
  2. Click the New... button to create a new instance
  3. Enter the following information and click OK button
    1. Instance Name:    StandAlone1
    2. Node:   localhost
    3. default-config
    4. Make a copy of the selected Configuration
  4. The table listed out this newly created instance.
  5. Select 'StandAlone1'  and click Start 
  6. The status refreshed once this instance is up and running

5. Application Deployment

User experience will be different than when there is only DAS.   

  1. Click the Applcations Tree node
  2. Notice that the Status column has changed.   It now says  "Enabled on All Targets".  
  3. Click Deploy... button.  Notice that you now can select any of the clusters and standalone instance as target.
  4. Select StandAlone1  and add that to the selected Target
  5. Click ok to deploy
  6. Click on 'Enabled on All Targets'  hyperlink
  7. Select 'StandAlone' and then  'Disabled' from the  '- More Actions- ' dropdown.  Notice that the status changed.
  8. Click on Manage Target... button to add or remove targets
  9. Add the ClusterABC  target
  10. Click on General tab,  examine the Modules and Components
  11. Click the Cancel button to get back to Applications listing
  12. Notice that the Status now says Enabled on 2 Target(s)

6. Monitoring through REST API

Like many of the pages, the monitoring statistics page has been converted to use REST API instead of through AMX.   However, due to issue# 12762,  you will need to add <monitoring-service> element in domain.xml to see the statistics showing up correctly.

  1. Click on Monitoring Tree Node to configure monitoring level
  2. Change Web Container level to HIGH,  click SAVE
  3. Deploy an application to "server",   I use stateless-simple.ear
  4. Go to Applications -> stateless-simple  tree node.  
  5. Click the launch link,  this will launch the application
  6. Go to Admin Server -> Monitoring tab
  7. Select this application and also TheGreeter component
  8. Note the statistic given.
