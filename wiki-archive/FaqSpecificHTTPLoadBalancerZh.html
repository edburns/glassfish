我要将 Web 服务部署到群集中。需要安装特定的 HTTP 负载平衡器插件吗？

由 users@glassfish 邮件列表改述：

*问：*我开发了一些 Web 服务，并使用 GlassFish v2 安装了一个包含两台计算机的群集。我要将我的 Web 服务部署到该群集中。我要将它们部署为 Web 应用程序（.war 文件）。需要安装 HTTP 负载平衡器插件吗？HTTP 负载平衡器插件只负责对作业在群集节点上进行负载平衡吗，或者如果我将 Web 服务部署到群集中，群集会自动对其进行平衡吗？

*答：*Alexis：群集和数据复制功能不需要特定的负载平衡解决方案。您可以使用 GlassFish 提供的软件负载平衡器（通过 GF 管理控制台对其进行管理），或使用任何第三方会话负载平衡（如 F5 的 BIG-IP，如 Prashanth 的博客文章中所述）。无论如何，您的群集"前面"多少都要有一定的负载平衡。

建议您阅读此文档：

  • Configuring the Cluster/Load Balancer with GlassFish V2（针对 GlassFish V2 配置群集/负载平衡器）
  • Clustering in GlassFish Version 2（GlassFish 版本 2 中的群集功能）

此外请听来自上届 JavaOne 会议的 GlassFish v2 演示，Technical Overview of GlassFish v2（GlassFish v2 技术概述）。
