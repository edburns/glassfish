ログファイルが見つからない

インストールに関する問題のトラブルシューティングには、以下に示すApplication Server のログファイルが役立ちます：

  • Server log file（サーバログファイル）: サーバの諸設定および配備に関する問題のトラブルシューティングに
  • HTTP server access logs（ HTTP サーバのアクセスログファイル）: HTTP サーバに発生する問題のトラブルシューティング、および Application Server のインスタンス内における HTTP リクエストのトレースに

インストールツールとアンインストールツールのいずれもログファイルを生成し、インストール／アンインストール時に発生する全てのイベントを記録します。これらのログファイルの最大の目的は、トラブルシューティングに役立つ情報を提供することにあります。
インストールツールのメッセージ、およびログファイルに加え、 Solaris の pkcinfo や showrev 、 Linux の rpm といったシステムユーティリティも、システムの情報を収集するのに役立ちます。

ログファイルの各エントリには、試行された操作と操作の結果、および、あれば失敗の原因が出力されます。

  • INFO: 正常に終了したインストール・タスクにはこのメッセージが付加されます。
  • WARNING: 致命的ではない障害にはこのメッセージが付加されます。一般に WARNINGメッセージ には障害の原因や性質、および考えうる解決策が含まれます。
  • ERROR: インストールやアンインストールの失敗を引き起こすような致命的なエラーには、このメッセージが付加されます。一般に ERROR メッセージ には、発生した障害の性質や原因に関する詳しい情報が含まれます。

なお、ドメイン固有のログは install_dir/domains/domain1/logs/ の中に出力されます。また、サーバのインストールに関するログファイルは一般に以下に出力されます：

  • Solaris へのインストール／アンインストール（ root ユーザ） : 
    /var/sadm/install/logs
  • Solaris へのインストール／アンインストール（ root ユーザ以外） : 
    /var/tmp
  • Linux へのインストール／アンインストール： 
    /var/tmp
