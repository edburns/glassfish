Development Update

The Milestone 4 document will provide clearer details of what will need to be tested and documented for Milestone 4.

Status of individual engineering efforts is as follows:

  • Monitoring:
    • Basic EE 6 functionality and v2 feature parity are to be implemented in Milestone 4.
    • TOIs to other teams are complete.
    • Uniform experience for all probe users is planned.
    • Schedule is aggressive.
    • End-to-end support for JDBC and web tier is expected on June 23.
    • GMBL registration is being added this week. Conflict resolution with web services is required.
    • The REST interface will be integrated in the trunk for milestone 4 with monitoring and configuration support. State management might not be in milestone 4. Security is to be addressed after milestone 4.
    • Some connector commands will be added and updated.
    • Adding Dtrace support is more involved than was originally thought.
    • Client side scripting and monitoring are also to be added.
  • Embedded GlassFish Server:
    • JAR files are to be added to the promotions.
    • Support for the Maven plugin is being added.
    • New embedded APIs are being done.
    • The use case document was reviewed by ASArch.
  • Configuration extensibility is being added. It affects the create domain task. Monitor Jerome Dochez's blog for more information. This information should be included in the Add-On Component Development Guide.

Open Mic/Issues
