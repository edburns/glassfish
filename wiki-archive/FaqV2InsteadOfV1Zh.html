我为何要使用 GlassFish V2 而不用 GlassFish V1？

GlassFish V2 除了添加了 GlassFish V1 中具备的完善的开发功能，还添加了如下所示的企业功能：

  • 群集（截屏视频 (screencast)）
    • 可伸缩性：动态增大或缩小群集大小以满足要求
    • 可用性：使用内存复制，可以将用户会话复制到多个节点。如果实例失败，则可以从另一个节点恢复会话数据而无需中断服务。
    • 单点管理：可以将群集作为单个实体来管理，提供单点部署和配置。
    • 设置群集极其简单。实际上，只需单击按钮，就可以将开发者配置文件升级到群集配置文件。（截屏视频 (screencast)）
  • 性能：GlassFish V2 可提供破纪录的性能。
  • 高级管理和监视
    • 群集是从单一位置进行管理的。从中央管理控制台这一个位置，即可创建群集、配置群集和使用应用服务器实例动态"填充"群集。
    • 可以从单一位置监视应用程序和容器。
    • 调用流分析可以在某一事务流经应用程序和各种应用服务器容器时跟踪该事务，同时捕获在各处所用的时间，以便于解决问题。
  • Web 服务器负载平衡器插件可供 Sun Java System Web Server、IIS 和 Apache 使用。此插件支持加权算法和循环算法。
  • Metro 项目－是一种高性能、可扩展、易于使用的 Web 服务栈。它是一站式服务，可满足您的所有 Web 服务需求，包括最简单的 hello world Web 服务和涉及 .NET 服务的可靠而安全的事务性 Web 服务。
