A.2 What build to use for FishCAT Testing ?

What are different GlassFish v3 Distributions click

What are the format of bit names in the build, b## stands for build number:

  • AppServer distribution:
    • zip format: glassfish-v3-ea-b##.zip
    • Unix installer: glassfish-v3-ea-b##-unix.sh
    • Windows installer: glassfish-v3-ea-b##-windows.exe
  • Web distribution:
    • zip format: glassfish-v3-web-ea-b##.zip
    • Unix installer: glassfish-v3-web-ea-b##-unix.sh

Please use the latest promoted build for FishCAT testing. The GF v3 promoted build is at: http://download.java.net/glassfish/v3/promoted/

Some time nightly build is used to verify a bug fix. The GF v3 nightly build is at: http://download.java.net/glassfish/v3/nightly/

A.3 How to log GlassFish bug

  • Must read. Check out the GuidelinesToCreateAnIssueInIssueTracker first.
  • Be sure that you are using the latest promote build
  • When you are not convinced whether a particular behavior is a bug or a feature, send a question to the quality [alias]
  • When you are convinced it is a bug, make sure it has not been filed, check V3 all open bug query
  • If you find it is a dup, please append your finding and add yourself to the interest list. When bug got fixed you will be notified to verify
  • To file a bug, GlassFish uses Issue Tracker to log bugs. Click Issue Tracker, click login and use your java.net ID to login.
  • Click New to open an issue
  • Select V3 for "Found in Version", select the correct "Subcomponent", select "Priority", P1/can't start the server, P2/lost data. P3/not too critical
  • Please fill in all the following details in the "Description": Which GlassFish V3 build, What JDK Version, What OS
  • Please give detailed steps to recreat, this is important information
  • Please attach server.log and the APPLICATION you have to log a bug first, then use attach feature
  • After bug got fixed, please verify it and then close or reopen if fix does not work

Questions about FishCAT program

B.1 What is the main purpose of FishCAT ?

The goal of this program is to get GlassFish community members involved in earlier testing of GlassFish V3. Participants will provide feedback on product usability, quality and performance. In return, the participants are given an opportunity to significantly influence and enhance the quality of GlassFish V3.

B.2 How do I participate FishCAT program ?

  • Please click Join and do step 1 and 2 to become a member of the GlassFish Quality Community.
  • Please apply for the FishCAT program by submit the FishCAT application form to glassfishquality@glassfish.java.net [Mailto]
  • We will contact you after you are selected to be FishCAT member
  • We will let you know when to start the FishCAT testing

B.3 What is the FishCAT task ?

As a member of the FishCAT program you will let us know two or more functionality areas you chose to test in GlassFish V3. In most cases, you need to run your application on weekly promoted build of GlassFish V3, occasionally, you will need to test nightly build of GlassFish V3 for bug verification purpose. You can use the issuetracker to report bugs and use quality@glassfish.java.net mailing list to report your experiences and suggestions on using the GlassFish V3. The main goal is to find as many bugs as you can, it is up to you on how you find bugs.

B.4 Do I have to concentrate on two areas in the application form?

You don't have to. You can test any area and file bug in any area you find. The reason we list functionality areas for you to chose so we can help to connect you to the developer and SQE in the areas. The contact person will help you to answer your question quickly or find the right place to address your questions. Another reason is to have balanced test coverage.
