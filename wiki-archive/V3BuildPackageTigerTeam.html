Goals:

  1. Improve build stability
  2. Define clean mappings between GlassFish V3 modules, packages, and distributions
  3. Define and implement automated processes for:
    1. Creating IPS enabled GlassFish distributions
    2. Building distributions on nightly and weekly basis (and probably also based on milestones and other release criteria)
    3. Distributing promoted artifacts to web sites and maven repositories as needed
  4. Create a complete source build for GFv3
  5. Define and implement a process for releasing other package formats
    1. Debian is the first and most important target here
    2. This may be done by converting IPS packages
