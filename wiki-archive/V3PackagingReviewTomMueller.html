GlassFish v3 Prelude File Layout and Packaging Review

Document: http://wiki.glassfish.java.net/attach/V3FunctionalSpecs/V3PreludePackagingFileLayout.txt?version=5

Reviewer: Tom.Mueller@Sun.COM

Review date: 17 July 2008

Response date: 28 July 2008

Item Type Comment Response
TM001 Clarification Is GFv3 intended to be integrated into OpenSolaris as a bundled component too? If so, would it have a file system layout for a bundled product, i.e., different from the one 4.5.1.1? If so, then there would have to be two sets of IPS packages for GFv3, with different package names. Has the package name selection in 4.5.1.2 taken this into account? This one-pager is intended to cover only GF v3 Prelude release which is not expected to be integrated into OpenSolaris as bundled component. Potential OpenSolaris distribution is post-Prelude deliverable and its packaging ramifications would be covered in separate specification for GF v3 full release.
TM002 Clarification I see there isn't a "3" in the package names. Is that intentional?  
TM003 Enhancement There are a set of IPS packaging best practices that we've collected. I recommend referencing this document from the one pager. Also, the one pager maybe should provide guidance as to the values for some common attributes (like version numbers, info.classification values, etc.)
