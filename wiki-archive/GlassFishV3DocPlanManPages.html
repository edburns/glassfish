Audience

The primary audience for the GlassFish v3 man pages is experienced system administrators who are managing the Application Server for an organization.

A secondary audience is developers who might be performing administration tasks, typically to enable testing of applications that are under development.

Purpose and Description

The GlassFish v3 man pages describe the Application Server administration commands, utilities, and associated concepts.

Related documents:

  • Administration Guide
  • Administration Console Online Help
  • Domain File Format Reference

Changes to Existing Man Pages

Some options are not supported at v3 Prelude. The descriptions of these options will be replaced with the following statement:

Do not specify this option. This option is retained for compatibility with other releases. If you specify this option, a syntax error does not occur. Instead, the command runs successfully and the option is silently ignored.

These options will not be removed from the man pages.

The following changes will be made to existing man pages:

  • Correct tagging and formatting errors (minor documentation impact)
  • Rewrite the descriptions of the following options that are not supported at v3 Prelude (moderate documentation impact):
    • --target in all man pages
    • --profile in all man pages
    • JavaDB options in man pages for deployment commands
  • State in the description of the --secure option in all man pages that the default is false (moderate documentation impact).
  • Modify create-jvm-options(1) to remove the -classpath-prefix, classpath-suffix, and -env-classpath-ignored
  • Modify generate-jvm-report(1) as follows:
    • To fix Issue 5162
    • To reflect the changes that were introduced by Issue 5803
  • Modify create-jdbc-connection-pool(1) to fix Issue 5766
  • Modify the following man pages to fix Issue 5837:
    • ping-connection-pool(1)
    • asadmin(1M) - need to add description of escape characters that were not carried forward from an earlier release.
  • Modify monitor(1M) to add descriptions of each argument of the --types option.
  • Modify create-domain(1) to reflect the changes that were intrdouced by Issue 5722.
