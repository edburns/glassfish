在高可用性群集中，"实例"是什么？

实例是 Java EE 服务器进程，可以将 Java EE 应用程序部署到这些进程中，然后从这些进程中提供 Java EE 应用程序。在群集环境中，会创建大量实例来帮助分散收到的请求负载（假设这些请求首先经过了负载平衡器或 IP 喷射器 (IP sprayer) 的处理），构建冗余以提供不间断的服务，而不会由于群集中一个或多个实例失败/崩溃而中断。换句话说，一组实例可以防止拓扑的软件层发生单点故障。

更进一步，会话状态复制可以提供不间断的服务和会话数据，这样便向站点的客户隐藏了软件进程/硬件故障。
