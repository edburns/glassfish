HowTo に関する質問

A.1 GlassFish に参加したばかりです。何から始めれば良いですか？

こちらに クイック・スタート・ガイド（英語） があります。これらのドキュメントは、 FishCAT メンバーが GlassFish を使い始める際にとても役立ちます。 GlassFish を使い始めたばかりの方は、 30 分ほど時間を取ってこれらのドキュメントを読んでみてください。 GlassFish のインストール方法やアプリケーションの配備方法、そして実行方法が分かることでしょう。

ドキュメントの Web サイトはこちらです : doc wiki （英語）

A.2 FishCAT におけるテストでは、どのビルドを使えば良いですか？

GlassFish v3 ディストリビューションの違いについては、 ここをクリック

各ビルド名のフォーマットは、 b## がビルド番号をあらわしています。

  • AppServer ディストリビューション：
    • zip 形式 : glassfish-v3-ea-b##.zip
    • Unix インストーラ： glassfish-v3-ea-b##-unix.sh
    • Windows インストーラ： glassfish-v3-ea-b##-windows.exe
  • Web ディストリビューション：
    • zip 形式: glassfish-v3-web-ea-b##.zip
    • Unix インストーラ： glassfish-v3-web-ea-b##-unix.sh
    • Windows インストーラ： glassfish-v3-web-ea-b##-windows.exe

FishCAT におけるテストでは、最新のプロモーションビルドを使用して下さい。 GF v3 のプロモーションビルドはこちらからダウンロードできます： http://download.java.net/glassfish/v3/promoted/

Some time nightly build is used to verify a bug fix.
バグ修正を確認するために、ナイトリービルドを使用する場合もあります。 GF v3 のナイトリービルドはこちらからダウンロードできます： http://download.java.net/glassfish/v3/nightly/

A.3 GlassFish のバグを記録するには

  • 必読： まずは 課題トラッカーへの課題登録に関するガイドライン（英語） をチェックして下さい。
  • 最新の プロモーションビルド を使用していることを確認して下さい。
  • When you are not convinced whether a particular behavior is a bug or a feature, send a question to the quality [alias]
  • When you are convinced it is a bug, make sure it has not been filed, check V3 all open bug query
  • If you find it is a dup, please append your finding and add yourself to the interest list. When bug got fixed you will be notified to verify
  • To file a bug, GlassFish uses Issue Tracker to log bugs. Click Issue Tracker, click login and use your java.net ID to login.
  • Click New to open an issue
  • Select V3 for "Found in Version", select the correct "Subcomponent", select "Priority", P1/can't start the server, P2/lost data. P3/not too critical
  • Please fill in all the following details in the "Description": Which GlassFish V3 build, What JDK Version, What OS
  • Please give detailed steps to recreat, this is important information
  • Please attach server.log and the APPLICATION you have to log a bug first, then use attach feature
  • After bug got fixed, please verify it and then close or reopen if fix does not work

Questions about FishCAT program

B.1 FishCAT の目的は?

本プログラムのゴールは、GlassFish V3 のテストに初期からコミュニティメンバーに参加してもらうことです。参加者は製品のユーザビリティ、品質、パフォーマンスに関してフィードバックを行います。その見返りとして、参加者には GlassFish V3 の品質に大きく影響を与えたり、もしくは品質を向上させたりする機会が与えられます。

B.2 FishCAT プログラムにどうやって参加するのですか?

  • こちらのページ のやり方に従って GlassFish Quality コミュニティのメンバーになって下さい。
  • 応募フォーム を [quality@glassfish.java.net] 宛てに送付し、FishCAT プログラムに応募下さい。
  • FishCAT メンバー選考の後、コンタクト致します。

B.3 FishCAT のタスクは ?

FishCAT プログラムのメンバーとして、あなたがテストする GlassFish V3 の 2 つの機能領域を選んで私たちにご連絡下さい。多くの場合、あなたは自分のアプリケーションを GlassFish V3 のウィークリープロモーションビルドで動かすことになります。たまに、バグの検証目的でナイトリービルドで動かすこともあるでしょう。問題の報告には issuetracker を使用し、v3 に関するあなたの経験や提案は quality@glassfish.java.net メーリングリスト宛てに報告して下さい。要はなるべく多くのバグを発見することが目的です。そのやりかたはあなたにお任せします。
