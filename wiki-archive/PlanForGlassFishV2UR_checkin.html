Hi All,

We are opening up the SJSAS91_UR1_BRANCH to approved check-ins.

__The check-in requirements will mandate

  • a separate person to review the changes,
  • PE and EE quick-look tests run with all tests passing,
  • a bug/RFE number
  • a short description of the change all documented in the check-in comments.
    • A bug number isn't enough to make quick sense of CVS logs for merging, please cut and paste the Synopsis/Summary of the related bug/RFE into the check-in comments at a minimum.

Please Note:

  • Soft code freeze is 10/31/07
  • Hard code freeze is 11/14/07
  • The SJSAS91_UR1_BRANCH will be merged back into the SJSAS91_FCS_BRANCH for inclusion into the Glassfish Sailfin release (9.1.1) at hard code freeze (11/14/07), so separate check-ins should not be required. The merge will be performed by the module leads.
  • If the check-in also pertain to the Trunk a separate check-in will be need to be completed there.

To submit bugs to be evaluated for inclusion into the SJSAS91_UR1_BRANCH:

Please send and email to Shreedhar (shreedhar.ganapathy@sun.com) and me (mark.basler@sun.com) with the following contents:

  • Bug/RFE number
  • Justification of why change is required
  • Scope of change
  • Impact to release
  • Confidence level that the fix won't negatively effect other components
  • Testing impact if adding any functionality
  • Schedule of when change will be committed

For this release there can't be any slip in the schedule because it will severely impact the Sailfin (9.1.1) release scheduled for beta (January 2008) and FCS (May 2008).

This release is scheduled over the Holidays, so please check-in your approved changes as soon as possible to avoid any last minute fires in your areas that could potentially interrupt the celebrations.

Please let us know if you have any questions or concerns...

Thanks for your cooperation - Mark & Shreedhar
