一般的な問題

このセクションでは、 GlassFish アプリケーションサーバを利用する際に直面しがちな、一般的な問題について解説します。

[

asadmin の start-domain コマンドが失敗する

再起動時にドメインやノードが自動的に再始動される

ログファイルが見つからない

ローカルサーバにアクセスできない (http://localhost:8080)

リモートサーバにアクセスできない

管理コンソールにアクセスできない

サーバアプリケーションにアクセスできない

Windows 上でサーバが始動しない(ポートの競合)

同一サーバ上で複数インスタンスをデバッグするとポートの競合が起こる

二つのサーバインスタンスが Windows 上の同一ポートにバインドされる

エラー： システムが指定されたパスを見つけられない

アプリケーションが persistence.support.JDODataStoreException エラーを生成する

asadmin の set コマンドが期待したとおりに動作しない

アプリケーション内で Jar ファイルに対してストリームを開いた後、配備解除や再配備が行えない (Windows のみ)

手動でディレクトリを削除すると、GlassFish アプリケーションサーバを再インストールできなくなる

サーバがクラッシュした後、 JVM のダンプを取れなくなる
