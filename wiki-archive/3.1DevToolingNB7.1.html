Introduction

When NetBeans 7.1 ships users will expect new features and bug fixes for the GlassFish Server integration. This project is supposed to make that happen.

Goal / Scope

The goal of this project is to expose more features of GlassFish Server that fit into the work-flow of the next new-feature release of NetBeans (post 7.0), which is being called 7.1 in this document..

A secondary goal is the continuous improvement of the integration between NetBeans and GlassFish.

Features

The priorities listed here are for the next 'new feature' release of NetBeans.

Feature ID Priority Description Engineering response Owner Estimated effort Source of Requirement Status/Comment
nbinteg-01 P1 Register a cluster yes vbk     need to extend the type of servers that can be registered as deployment targets to include clusters
nbinteg-02 P2 register a stand-alone instance maybe vbk     registering a stand-alone instance seems redundant from a developer perspective, since the DAS is a stand-alone instance in most every respect
nbinteg-03 P1 start a cluster yes vbk    
nbinteg-04 P2 start a cluster with the instances in debug mode maybe vbk     depends on an extension of the start-cluster command.
nbinteg-05 P1 stop a cluster yes vbk    
nbinteg-06 P2 stop a stand-alone instance maybe vbk     won't support if start is not supported
nbinteg-07 P1 view admin console yes vbk     open the admin console for a registered cluster
