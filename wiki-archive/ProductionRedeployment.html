Check In Process

1. Request code review from relevant GlassFish teams and incorporate comments . Code review comments would normally be provided within 48 hours.

2. New unit tests/functional tests are required for new functionality.

3. QuickLook, deployment dev tests are required to run before each check in. Additional tests (like web dev tests and ejb dev tests) may be required for some check ins to ensure no regression. The GlassFish team will communicate any additional test suites that should be run at code review time.
