GlassFish v2 与 Sun GlassFish Enterprise Server v2 有何不同？

GlassFish v2 是由 GlassFish 社区提供的可立即用于企业的 Java EE 5 应用服务器。成熟的代码库是 Sun 将 Sun Java System Application Server 8.x 贡献给该社区的结果。严格来讲，该社区由 Sun 领导，当前正在制定管理方法。Oracle 和 Ericsson 也是主要贡献者，分别贡献了 Toplink Essentials 和 SIP Servlet 技术。

Sun GlassFish Enterprise Server v2 是 Sun GlassFish v2 的商业版本。

<SUP>(1)</SUP> 可以免费下载这些包中的任何一个并将其部署到生产中。<BR><BR>
<SUP>(2)</SUP> 第三方图形安装程序。Sun 趋向支持对 GlassFish v3 使用开源安装程序。<BR><BR>
<SUP>(3)</SUP> 可在生产中免费使用 HADB。HADB 可提供 99.999% 的可用性，但需要额外设置、配置和管理。对于支持，提供了 HADB 订阅，需要等效的 Sun GlassFish Enterprise Server 订阅。<BR><BR>
<SUP>(4)</SUP> HADB 即高可用性数据库 (High Availability Database)。HADB 提供 99.999% 的可用性，它是用户会话的分布式持久性库。HADB 不是开源的，但可在生产中免费使用。Sun 认为 95% 以上的部署都会使用更易用且强健的内存会话复制。对于现有的客户，如果该客户正在进行有关 HADB 的有效处理过程，Sun 将继续提供支持。如果社区成员对 HADB有足够的兴趣，Sun 可以重新考虑不开放 HADB 源代码的决定。<BR><BR>
<SUP>(5)</SUP> 随 GlassFish Enterprise Server 附送的 Data Direct JDBC 驱动程序可免费使用（无论已订阅还是未订阅）。只有 GlassFish Enterprise Server with HADB 包中提供 Data Direct JDBC 驱动程序，因此您需要下载该包。发行说明中列出了 Data Direct 驱动程序。订阅 GlassFish Enterprise Server 可以提供对驱动程序的生产支持。<BR><BR>
<SUP>(6)</SUP> HADB 仅随 GlassFish Enterprise Server w/HADB 包提供。其他包仍然可以利用 HABD，但需要 "w/HADB" 包才能安装 HADB。使用案例：用户下载用于评估的 GlassFish 或 GlassFish Enterprise Server 包。用户以后决定尝试 HADB，但希望利用现有的安装。用户必须下载 "w/HADB" 包以安装 HADB。
<SUP>(7)</SUP> GlassFish v3 和 GlassFish Enterprise Server 的下一个主要发行版将共享通用开源安装程序。
<SUP>(8)</SUP> 多亏了 Julien Ponge，GlassFish V2 UR1 的 lzPack 图形安装程序才可用！
