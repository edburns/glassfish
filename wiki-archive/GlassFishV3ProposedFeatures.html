Proposed Features for GlassFish V3

Serviceability

  • Make it easier for support organization to provide top quality support.

Modularization

  • Modularization of the AppServer

Web.Next

  • Additional Web 2.0 Features, possibly including ProjectPhobos, JMaki, and others.

Productivity for Developer Team

  • Very easy to replace a single module and test it (epll).

Productivity for Developers

  • very fast startup time
  • possibly embeddable (i.e could run in the same VM as an IDE).
  • Online documentation - just add asadmin and query for any documentation. Learn from emacs. (epll)

Domain Configuration Management

  • Dynamic configuration (epll, but kedar should own)

per application resource management instead of global

  • sun-resources.xml support cleanup

Lower port "footprint"

  • use fewer ports
  • select more ports dynamically, based on the current "port environment" instead of just failing due to port conflicts
