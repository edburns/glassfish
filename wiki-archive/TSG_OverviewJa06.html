エラーメッセージ

何らかの問題には、アプリケーションの正常な動作が妨げられていることを示しているエラーメッセージが付きものです。

  • いくつかのケースでは、メッセージは、何が間違っているのか、また、対応(どちらかというと解決)のために何が必要かということに関して、とても判りやすいものになっています。たとえば、もしあなたが asadmin start-domain コマンドを使用してドメインを起動させて、ドメイン起動後にうっかりして同じコマンドを再度実行した場合、以下のようなメッセージが表示されます:\\\\\~UWC_TOKEN_START~1278691907248~UWC_TOKEN_END
    \このケースでは、メッセージは明らかなガイダンスと問題は無視できるものであることを示しています。
  • 時折、エラーメッセージが、問題または解決方法についてのおおまかな情報のみを表していたり、複数の可能性に関して示唆していることがあります。たとえば、以下のような例があります:\\\\\~UWC_TOKEN_START~1278691907249~UWC_TOKEN_END
    \このケースでは、問題は明確ではありません。または、複数の誤りがある可能性があります。あなたは、さまざまな可能性と、もしかしたらいくつかの解決策に関して、よく考える必要があるかもしれません。もし、(エラーメッセージが示している)解決策が時間やコストが掛かるものであれば、その解決策が正しいものかどうか、実際に何かを行う前に確認するステップを踏むようにしてください。
  • いくつかのエラーメッセージは、有効ではなく、かつ、提供する助言的な情報も少ないものです; たとえば：\\\\\~UWC_TOKEN_START~1278691907250~UWC_TOKEN_END
    \このケースでは、何か対応するにしても、とても少ない情報しかありません。エラーを引き起こした実際の状況を確認して、(エラーの発生前に)どのような兆候があったのかを確認することが特に重要です。

GlassFish アプリケーションサーバのすべてのエラーメッセージの説明は、Sun Java System Application Server 9.1 Error Message Reference Wiki を参照してください。
