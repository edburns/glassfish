休止終了後のサーバーレスポンス速度の低下

  • 解説
    \長い休止状態の後、サーバーは、サービスがリクエストを受け付けるのに、長い時間を要し、そして、サーバーログの書式に "lost connection" と言うメッセージが見つけられます。 :\\\\\~UWC_TOKEN_START~1278691906829~UWC_TOKEN_END
    \このようなケースでは、サーバーに、 HADB に対する JDBC プールを再作成する必要があります。
  • 解決方法
    \タイムアウト値を変更してください。デフォルトの HADB 接続タイムアウト値は、 1800 秒です。もしもアプリケーション・サーバーが、 JDBC 接続終了前にリクエストを送信しなかった場合、 HADB は接続を遮断し、アプリケーション・サーバーは、接続を再構築する必要があります。タイムアウト値を変更するには、 hadbm set SessionTimeout= コマンドを使用します。
    留意事項 : Make sure the HADB 接続タイムアウト値が、 JDBC 接続プールタイムアウト値よりも大きいことを、必ず確認してください。もしも JDBC 接続タイムアウト値が、 HADB 接続タイムアウト値よりも大きい場合、接続は、 HADB 側から遮断されますが、アプリケーションサーバーの接続プールは、そのまま残存します。アプリケーションが接続の利用を試行したとき、アプリケーション・サーバーが、接続を再作成すると言う大きなオーバーヘッドを招きます。
