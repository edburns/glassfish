Audience

The primary audience for the GlassFish Administration Console is system administrators who are managing GlassFish Server for an organization.

A secondary audience is developers who might be performing administration tasks, typically to enable testing of applications that are under development.

Summary of the Feature From a User's Perspective

The Administration Console is one of the tools for application server administrators to configure and monitor the application server.

Availability

This feature is available in both the web and full GlassFish Server distributions.

For each module, the help provides two types of topics:

  • Reference Topics. The context-sensitive topics displayed when a user clicks the Help button while viewing a particular Administration Console page. These topics describe the fields and other user interface elements on the page.
  • Task Topics. Topics that tell administrators how to perform common tasks, even if performing the task requires navigation across Administration Console pages.

The following tables list the reference and task topics for each online help module. For each topic, the following information is provided:

  • Topic Title. The title of the topic.
  • Topic Id. The id of the topic in the online help system.
  • Doc Impact. The documentation impact on the topic arising from 3.1 product development.
  • Writer. The writer assigned as the content owner of the topic. This writer is responsible for making any content changes to the topic and for reviewing (or assisting with) any inline help changes needed on the topic's corresponding Admin Console page.
  • Reviewers. Reviewers assigned to review any content or organizational changes to the topic.
  • Description of Change. A description of the change or changes to the topic. Because changes to a topic can be driven from different GlassFish projects, changes are introduced by an abbreviation that denotes the project driving the change:
    • ACC: App Client Container
    • AdCon: Management Console
    • AMX: JMX API (AMX)
    • CDI: Contexts and Dependency Injection (CDI)
    • Clus: Clustering Infrastructure
    • CPRJ: Connectors, Pooling, Resources, JDBC
    • Depl: Deployment
    • EJB: EJB
    • Emb: Embedded
    • File: File Layout
    • G11n: Globalization
    • GMS: GMS Configuration and GMS Runtime Clustering
    • IIOP: IIOP
    • IMSR: In-memory Session Replication
    • Inst: Installer
    • Jersey: Jersey
    • JMS: JMS Integration
    • JSF: JSF
    • L&D: Logging & Diagnostics
    • LB: Loadbalancer
    • Metro: Metro

    • Pack: Packaging
    • Pers: Persistence
    • REST: RESTful API
    • Sec: Security
    • SSH: SSH/SSHD
    • Trans: Transaction Service
    • Upgr: Upgrade
    • WebT: Web Tier
