2. Discussion: approaches to providing documentation for multiple distributions

  • Discussed the three options listed on the wiki page.
  • Eliminated option #3 because it doesn't scale well.
  • Discussed options #1 and #2; decided on option #2: one doc set for all distros.
  • Paul will present this recommendation to the product team, also describing the other options that were considered and why they were rejected.
  • User confusion will be minimized by providing doc centers/roadmaps and text labels:
    • If a feature is available in all distros, no text label is needed.
    • If a feature is unique to a specific distro, a text label must be used, for instance, "Perform this procedure only if you have x distribution" or "This command is available only if you have x distribution."
  • Marked sections won't be used for different distros.
  • One IPS package will be provided that covers all distros (no distro-specific IPS packages).
    \

3. Progress on writing assignments by feature

  • In future meetings we'll go through the list feature by feature; feature owners will be asked to provide status; we'll also track milestones such as task analysis scheduling, doc plan creation and review, etc.
  • SMEs not ready to discuss the specs due to competing priorities.
  • Not all specs are available; in the meantime, we should review those specs that are available, schedule task analyses, start docs plans, look at related information in the existing doc set (to identify which bits of the doc set will be affected), etc.
