GlassFish マイグレーションガイド

本マイグレーションガイドの目的は、GlassFish へのアプリケーションの移行をお助けすることです。

どう執筆するか。

  • GlassFish Wiki ID でログイン (上部の "Log in" ボタンから) して下さい。この ID は java.net ID とは別ものです。もしまだ GlassFish Wiki ID をお持ちでない場合には、ログインページ、"Register New User" タブの説明に従って、ご登録下さい。
  • 適切な節の下に、あなたの執筆する移行に関する課題、解決策のページを新規に作成下さい。

JBoss -> GlassFish 移行上の課題

  • 標準でない JNDI 名を使用するローカル EJB 参照
  • WAR ファイル最上位にパッケージされるクラス
  • Facelets の利用
  • JBoss EJBQL != Glassfish(J2EE Spec) EJBQL
  • GlassFish Cache でエンティティとして参照される組み込みオブジェクト
  • GlassFish キャッシュリフレッシュ

WebSphere -> GlassFish 移行上の課題

  • web.xml 内 <servlet-mapping> での不正な URL パターン

WebLogic -> GlassFish 移行上の課題

  • JSP ページでのパッケージのインポート

Tomcat -> GlassFish 移行上の課題

JaxRPC -> JAX-WS 移行上の課題

  • Fast Infoset コンテントネゴシエーション (英語)
