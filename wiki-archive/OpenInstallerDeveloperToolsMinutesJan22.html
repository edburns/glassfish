Meeting minutes

  • Gerard briefed about the initial plan of the project and talked about high level workflow.
  • James talked about the following
    • Media location is used as the reference in a openInstaller based installer.
    • We need other files too such as page sequence to build a complete installer.
    • Unique port validation in openInstaller is still being worked upon.
    • The output XMLs can be zipped to feed into the assembly tool.
    • Nice To Have: view visual panels being shown from the tool. There are tools available for view which help to do this (web page based preview).
    • Multiple views can be defined for a single data model file (model).
  • Gerard mentioned about different use cases being considered.
  • There can be plugins or extensions to this tool for any custom task.
  • Import feature: import files as references (read only) or include them in the project? Decided to have both.
  • Export feature will export the files which are included as part of the project and not the references.
  • Config wrappers will not be in the scope of this tool.
  • TO DO - to generate use cases, screen mock ups and storyboards for different scenarios.
  • We need to find a way to include Best practices and Brand specifics as add ons to this tool.
